## 4. Recommendations for SDDs

1. **SDD-001**: Audit the Capsule chart `values.yaml` before rendering. Confirm: `--log-level` is not `debug`, no secrets embedded, controller requests sane resources, no wildcards in `additionalRoleBindings`. Document findings in the Work Log.
2. **SDD-001**: Configure `replicas: 2` for the Capsule controller in `values.yaml` to avoid single-pod DoS; verify this is compatible with the chart v0.12.x schema.
3. **SDD-001**: Set `timeoutSeconds: 5` (or lower) on the validating webhook configuration via a `customPatch` in the kustomize folder â€” prevents slow webhook from becoming a cluster-wide DoS surface.
4. **SDD-001**: Pin the Capsule controller image by digest in the rendered manifest (override from `values.yaml` or post-render patch). Acceptable to defer but must be tracked as a TODO in the Work Log and surfaced as a finding.
5. **SDD-001**: Add a comment header to the rendered `capsule-from-helm.yaml` containing chart version and regen timestamp; the Makefile must emit these automatically so the diff is self-documenting.
6. **SDD-002**: Verify `plugins.kustomize` entries run AFTER the networking module during `furyctl apply` (Cilium must be Ready before Capsule webhook is deployed). Document the observed ordering in the SDD Work Log.
7. **SDD-003**: Add a BATS test that creates a Tenant CR attempting to bind `cluster-admin` via `additionalRoleBindings`; assert Capsule rejects it or at least does not propagate the binding into tenant namespaces.
8. **SDD-003**: Add a BATS test that confirms the `ValidatingWebhookConfiguration` named `capsule-validation.projectcapsule.dev` (or equivalent) exists after install, has `failurePolicy: Fail`, and references a webhook with `timeoutSeconds <= 5`.
9. **SDD-003**: Add a BATS test that applies a Tenant CR with a sane quota (e.g., 3 namespaces, 1Gi storage), creates a namespace owned by the tenant, and verifies the `ResourceQuota` is attached.
10. **SDD-003**: Use `setup_file`/`teardown_file` to create+delete a uniquely-named test Tenant; never reuse names across tests; always run teardown even on failure.
11. **SDD-004**: Extend `tests/05-security.bats` to assert that the webhook configurations for Capsule are present and that no rogue `ValidatingWebhookConfiguration` pointing outside `capsule-system` exists.
12. **SDD-004**: In `mise run all`, add a final guard that confirms the cluster context is still `kind-fury-baobank` before/after Capsule install â€” avoids accidentally applying the Tenant test fixtures elsewhere.
