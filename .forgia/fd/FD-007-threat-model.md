| No threats identified | **T** (Tampering) | — | AppRole credentials are read-only from the VM's perspective. |
| No threats identified | **R** (Repudiation) | — | AppRole login is logged in Vault audit with client IP. |
| No threats identified | **D** (Denial of Service) | — | Failed auth → agent retries. Doesn't affect other nodes. |

## 4. Recommendations for SDDs

1. **SDD-001** (sysext build): Verify SHA256 checksum of downloaded OpenBao binary against GitHub release checksums file. Store expected checksum in the build script. BATS: `openbao version` on the VM matches expected version. For future: consider signing the sysext image.

2. **SDD-002** (Ignition/Butane): Set file permissions in Ignition — `agent.hcl` at 0600, SSH key only for lab (document as non-production). VAULT_ADDR must be parameterized (not hardcoded). BATS: verify agent.hcl points to expected OpenBao endpoint.

3. **SDD-003** (AppRole): Use wrapping token with `wrapping_ttl=5m`. Do NOT pass wrapping token as QEMU command-line argument (visible in ps). Use fw_cfg or file-based delivery instead. Set `token_ttl=1h`, `secret_id_num_uses=1`. BATS: verify the wrapping token has been consumed (second unwrap attempt fails).

4. **SDD-004** (QEMU launcher): Teardown script must delete the VM disk image (contains secrets). Use QEMU fw_cfg for wrapping token delivery. Set up port forwarding: host:2222→VM:22 (SSH), host:8200→OpenBao. BATS: verify disk is deleted after `vm-down`.

5. **SDD-005** (BATS + wiring): SSH-based tests must verify: file permissions (0600 for keys), cert content (CN, O, SAN), agent service status, systemd dependency (kubelet After openbao-agent). Include a "secret exposure" test: verify no wrapping token or secret_id remains on disk after boot.
