## 4. Recommendations for SDDs

1. **SDD-001**: Pin the `kindest/node` image by digest in `.zlab.yaml` (not a tag like `v1.31.0`). Document the upstream source of the digest and the rotation procedure. Ensure worker node labels (`node-role.fury.io/infra=true`) do not grant additional privileges — labels are for scheduling only.

2. **SDD-001**: In `.zlab.yaml` port mappings, bind to `127.0.0.1` (loopback) only — never `0.0.0.0` — so Kind ports are not reachable from the LAN.

3. **SDD-002**: Pin the exact git SHA of `fury-kubernetes-networking` in `Furyfile.yaml`. Do not use branch names or floating tags. Document how to verify the SHA matches an upstream signed release.

4. **SDD-002**: In `furyctl.yaml`, use `customPatches` to pin every Cilium and Hubble container image by SHA digest, overriding any `:vX.Y` tags from the module defaults.

5. **SDD-002**: Ensure every `furyctl` invocation uses `--disable-analytics`; add this flag in the `mise run` task definitions, not as a manual step.

6. **SDD-002**: Verify the KFD networking module kustomization does not mount the Docker socket into any cluster pod. If it does, override via `customPatches` to remove the mount — it is not needed for CNI operation.

7. **SDD-003**: Expose Hubble UI via port-forward only in the default `mise` task. If a NodePort option is added, document it is lab-only, bind it to `127.0.0.1` via Kind port mapping, and do not add it to the `mise run up` default path.

8. **SDD-003**: Enable Hubble mTLS via `customPatches` if the upstream KFD networking module does not enable it by default. Validate via BATS test that the Hubble relay rejects unauthenticated gRPC calls.

9. **SDD-004**: Add a BATS test that asserts `kubeconfig` file permissions are `0600` after cluster creation. Fail the E2E test if the permissions are more permissive.

10. **SDD-004**: Add a BATS test that asserts the cluster name/context matches `fury-baobank` (or the equivalent set in `.zlab.yaml`) — prevents accidental production kubeconfig use.

11. **SDD-004**: Add a BATS test that asserts no pod in `kube-system` beyond Cilium agents has `privileged: true` or CAP_NET_ADMIN. Fail if an unexpected privileged pod exists.

12. **SDD-004**: Add a BATS test that asserts Hubble UI is NOT reachable on `0.0.0.0` — only on `127.0.0.1` — to catch accidental exposure.
